# Cafe Sales Analysis

> Очистка «грязных» данных и разведочный анализ продаж кафе на Python

## О проекте

Проект посвящён работе с намеренно «загрязнённым» датасетом продаж кафе. Исходные данные содержат ошибки типа `ERROR` и `UNKNOWN`, пропуски в числовых и категориальных столбцах, а также некорректные типы данных. Основная задача — привести данные к пригодному для анализа виду с минимальными потерями строк, а затем выявить закономерности в продажах.

---

## Цели и задачи

- Очистить данные, заменить значения-заглушки, привести типы данных
- Восстановить пропущенные значения через логические зависимости между столбцами
- Оценить качество данных и задокументировать принятые решения
- Провести EDA, выявить топ-товары, временны́е паттерны и сезонность

## Этапы анализа

**Часть 1 — Очистка данных (Data Cleaning)**
1. Загрузка данных и первичный осмотр
2. Приведение названий столбцов к `snake_case`
3. Очистка числовых столбцов (`ERROR` / `UNKNOWN` -> `NaN`, конвертация в `float`)
4. Очистка и преобразование дат -> `datetime`
5. Очистка текстовых столбцов
6. Проверка на дубликаты и оценка пропусков
7. Проверка целостности `total_spent = quantity * price_per_unit`
8. Восстановление пропусков через арифметическую зависимость
9. Удаление невосполнимых пропусков
10. Восстановление названий товаров через уникальную цену
11. Заполнение пропусков в категориальных столбцах

**Часть 2 — Разведочный анализ (EDA)**

12. Проверка распределения по датам
13. Ключевые бизнес-метрики (выручка, средний чек)
14. Топ-товары по количеству продаж и по выручке
15. Выручка по дням недели
16. Выручка по месяцам
17. Сезонный анализ

## Ключевые выводы

- **Кофе** — самый популярный товар по количеству продаж (3897 единиц), однако не входит в топ-5 по выручке
- **Салат** — лидер по выручке, что подчёркивает разницу между популярностью и маржинальностью
- **Осень** — самый прибыльный сезон (25251), **зима** — самый высокий средний чек (9,04)
- **Четверг, пятница и воскресенье** приносят наибольшую дневную выручку
